Помощь по интерфейсу администрирования

Основные понятия

Данная система позволяет организовывать события. Каждое событие характеризуется в первую очередь набором соревнований, которые проводятся в его рамках, и набором зарегистрированных на событие участников. События имеют идентификатор, (примеры идентификаторов: bbtc, bbtc-2013, ...), идентификатор можно увидеть в начале адресной строки в браузере, например, все адреса события bbtc начинаются с http://on-line.runodog.ru/bbtc/.

Одновременно можно проводить несколько событий. В данный момент созданы события bbtc, bbtc2, bbtc3. Первое хранит информацию о конкурсе, проведенном в мае 2013, на него до сих пор могут приходить участники, поэтому с этим событием нужно работать аккуратно. Другие события можно использовать для тестирования системы и для подготовки новых событий. При необходимости получить дополнительные события, обращайтесь к И. Посову

Каждое событие состоит из нескольких соревнований. У соревнований есть наборы заданий, время начала, время окончания, и другие параметры.

Система имеет два вида пользователей: участники и администраторы. Участники регистрируются на события, участвуют в соревнованиях, администраторы настраивают события и соревнования, и тоже могут участвовать в соревнованиях в целях проверки.

Настройка события

Для настройки события администратор должен выбрать слева пункт "Администрирование". Внутри доступны следующие настройки:

1. Название события. Название события отображается в заголовке на каждой странице события.

2. Дата отображения результатов. После этой даты все участники будут видеть свои результаты по событию, в частности это суммарный балл за все соревнования и занятое место.

3. Дата начала регистрации. Пока эта дата не наступит, участники не смогут зарегистрироваться на соревнование, они будут видеть текст, что регистрация еще не началась.

4. Дата завершения регистрации. После этой даты участники не смогут зарегистрироваться на соревнование, они будут видеть текст, что регистрация уже закончилась.

5. Список школ. Список школ - это набор кодов школ. Эти коды можно необходимо указывать при регистрации. Загружать необходимо файл в формате CSV, содержащий один столбец с заголовком в первой клетке.

6. Данные по событию. Этот раздел содержит ссылки на скачивание таблиц с информацией о событии, например, с информацией о зарегистрированных пользователях. Иногда ожидание перед загрузкой файла может затянуться, особенно, если количество участников большое.

Список соревнований

Список соревнований доступен администратору при выборе пункта "Соревнование" в меню слева.

Управлять списком соревнований можно, добавляя, удаляя и перемещая соревнования. Кнопки удаления и перемещения доступны в левом нижнем углу прямоугольника с соревнованием.

Добавление соревнования доступно внизу страницы. Необходимо указать название соревнования и его идентификатор. Идентификатор - это произвольный набор латинских букв и цифр, его предназначение будет указано позже. В качестве примера, в событии bbtc для экспресс-теста используется идентификатор express, а для теста повышенной сложности идентификатор hard.

Внизу блока каждого соревнования находится дополнительно ссылка на скачивание результатов соревнования, и находится ссылка "администрирование соревнования", позволяющая изменять настройки соревнования.

Настройка соревнования

После перехода по ссылке "администрирование соревнование" для одного из соревнований, появляется страница со следующими возможностями.

1. Информация, данные о соревновании. В этом разделе указано, сколько людей начали участие в этом соревновании, и находится ссылка на скачивание результатов соревнования.

2. Название соревнования, название, которое отображается для всех пользователей.

3. Дата начала. Начиная с этого момента участники могут приступать к соревнованию. Эта дата не распросраняется на администраторов, они способны приступить к любому соревнованию в любое время.

4. Дата окончания. После этого момента участники не могут приступать к соревнованию. Эта дата не распросраняется на администраторов, они способны приступить к любому соревнованию в любое время.

5. Дата отображения результатов. После этого момента участникам становятся доступны резульаты по данному соревнованию. Дата может быть не указана, тогда она совпадает с соответствующей датой по всему событию. (см. выше). Если дата отображения результатов указана раньше, чем дата начала соревнования, то считается, что результаты доступны участникам сразу.

6. Минут на прохождение. Именно столько минут отведено участнику на прохождение. Ноль означает, что время не ограничено.

7. Описание. Описание соревнования видно участникам в блоке с информацией о соревновании.

8. Размеры страниц. Описывает, сколько задач во время решения задач показывается на одной странице. Число должно быть указано в квадратных скобках. Если разные страницы имеют разные размеры, то их размеры должны быть перечислены через запятую, например [5,7,5,5].

9. Таблицы с данными. Описывает содержимое таблиц с информацией о результатах соревнования. Изменять здесь скорее всего ничего не понадобиться, поэтому подробного описания не будет. Этот пункт настроек позволяет менять местами столбцы таблицы или изменять их название.

10. Можно перезапускать. Позволяет участникам повторно проходить одно и то же соревнование, сбрасывая его результаты. Это удобно для тренировочных соревнований. Администраторы имеют возможность перезапускать любое соревнование всегда, независимо от того, установлена ли здесь галочка.

11. Только для администраторов. Эта галочка позволяет сделать так, что соревнование видно только администратору и не видно участникам. Если галочка установлена, то администратор при просмотре списка соревнований под названием соревнования видит фразу "(Только для администратора)".

Загрузка заданий

Задания в системе структурированы как файлы на диске, они хранятся в папках, которые в свою очередь могут храниться в других папках. Используется следующая структура каталогов, рассмотрим ее на примере соревнования с идентификатором bbtc. В корневой папке bbtc находятся все задания соответствующего события. В подпапках bbtc/test, bbtc/express, bbtc/hard, bbtc/explore хранятся задания соответствующих событий. Названия подпапок соответствуют идентификаторам соревнований. В подпапках с числовыми названиями хранятся задания из блоков, например, в bbtc/express/6 хранятся задания из шестого блока экспресс-теста "Артикли и предлоги". Сами задачи имеют названия типа RoundId: bbtc/express/6/A147, bbtc/express/7/A172 и т.п.

Для загрузки заданий используется раздел "Загрузить задачи для соревнования" на странице администрирования соревнования. Формат файла с задачаниями был зафиксирован заранее, это CSV файл со столбцами: ID, RoundID, Block, BlockName, BlockID, Group, Question, AnswerA, AnswerB, AnswerC, AnswerD, AnswerE, Correct, Answer, score_right, score_wrong, score_skipped, Statistic, Comment.

После загрузки задачи попадают в папку соответствующего соревнования, эта папка написана рядом с фразой "Загрузить задачи для соревнования", например, "(в /bbtc/express)". Они раскладываются по папкам блоков, номер блока берется из столбца Block, название задачи берется из столбца RoundId.

Назначение заданий соревнованию

После того как задачи загружены, необходимо указать, какие задания использовать в соревновании. Задания соревнования состоят из нескольких блоков. Каждый блок может иметь один из трех видов. Либо блок - это одно задание, либо блок - это все задания из определенной папки, либо блок - это несколько случайных заданий из определенной папки.

Блоки перечислены на странице администрирования соревнования в разделе "Наборы заданий".

Справа от каждого блока находится стрелка вверх, вниз или крестик, они позволяют, соответственно, изменять порядок блоков или удалять их.

Для добавления блока используется раздел "Добавить блок". Возможные варианты строки конфигурации на примерах:

5 random <- /bbtc/express/6 означает, что необходимо взять 5 случайных заданий из блока 6 экспресс теста.

problem /bbtc/express/6/A147 означает, что блок состоит из одного задания A147.

folder /bbtc/hard/6 означает, что нужно взять все задания из 6-ого блока теста повышенной сложности.

Добавление пользователей

Добавление пользователя может быть необходимо, если регистрация закрыта, а хочется посмотреть на событие от имени обычного участника. Далее процесс добавления пользователей будет значительно упрощен.

Чтобы добавить пользователя, нужно быть залогиненным от имени пользователя с правами администратора и перейти по ссылке http://on-line.runodog.ru/[event id]/[login]/[password]/[email]/[1 for admin]/create_user, например http://on-line.runodog.ru/bbtc/ilya/qwerty/iposov@@gmail.com/1/create_user создаст пользователя с логином ilya, паролем qwerty, почтой iposov@@gmail.com, 1 - означает, что он администратор. 0 означало бы, что он обычный пользователь.
